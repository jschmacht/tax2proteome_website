FAQ

How should I assemble my database?

You should know your sample composition or make a good estimate. With known taxon IDs you can use tax2proteome to select all protein sequences from the public protein databases from NCBI or UNiProtKB. Taxon IDs can be determined using the NCBI Taxonomy website (https://www.ncbi.nlm.nih.gov/taxonomy) or using the search field.
For including the determined taxa use one of the following options or a combination of them:
font_download Text search with name or taxon ID.
view_list Enter multiple taxon IDs seperated by space, comma or line break.
note_add File Upload (extracts all contained numbers). Only .csv, .tsv and .txt files allowed.

Depending on your analyzed sample, you can choose one of the public protein databases and a taxonomic rank. See also FAQ Which database should I use? and Which taxonomic rank should I use? .

Which database should I use?

The most important public protein databases are the UniProt Knowledgebase (UniProtKB) database (https://www.uniprot.org/uniprot/) (2019) and the National Center for Biotechnology Information (NCBI) non-redundant database (https://www.ncbi.nlm.nih.gov/protein).

NCBI-nr
The NCBI non-redundant database is a collection of sequences from several sources including UniProtKB. It is the biggest public protein database with approximately 220 million entries (database state 09.09.2019).

UniProt
The UniProtKB database is composed of the Swiss-Prot and TrEMBL databases and contains about 172 million sequence entries (database state 09.09.2019). The combination of both is called Uniprot by this tool. The protein sequences originate from translations of coding sequences (CDS) submitted to the EMBL-Bank/GenBank/DDBJ nucleotide sequence resources.

SwissProt
Swiss-Prot is the manually curated section of UniProtKB with minimal redundancy and contains about half a million fasta entries (database state 09.09.2019). It is recommended as resource for microbial sequences only for model organisms.

Recommandation
Important for your database choice is the size of the expected database. If your sample contains many different organism and/or model organisms with huge database representation, we recommend to use the UniProt database. In our example samples, the UniProt selected databases contained up to 40 % less sequences, which improves the identification rates. If the databases size becomes too large, the identification rate will decrease. For rare species the NCBI-nr database may contain more relevant sequences.

Which taxonomic rank should I use?

Only a small proportion of organisms is fully sequenced so far. Currently only for model organisms or medically interesting species public protein sequence databases can be considered reasonably representative. If your organisms belong to one of these, you should use the rank ‘species’. If the analyzed species are poorly represented in the public protein databases, increasing the rank (e.g. to 'genus' or 'family') may increase the sequence coverage by including sequences of related species. However, this should be carefully considered, since an increase in the database size is often accompanied by a decrease in the identification rate. The entire bacterial superkingdom database (taxon ID = 2) from Uniprot or NCBI-nr is too large for the most search engines and should never be used. The more specific the database is and the better the sequence coverage, the better are the spectral matching results.

What is the difference between 'reduced headers' and 'full headers' options?

This option is only available for NCBI-nr selected databases to reduce the database size. If the ‘reduced headers’ option is chosen, the often enormous headers of NCBI non-redundant database, which contain all redundant sequence headers concatenated, are reduced to their accession IDs. This reduces the memory requirements for the generated database by up to 50 % or more.
Separator of different accession IDs: SOH (\x01)
An assignment of the accession IDs to their corresponding taxa is possible using the prot.accession2taxid and pdb.accession2taxid files from NCBI.
ftp.ncbi.nlm.nih.gov/pub/taxonomy/accession2taxid/prot.accession2taxid.gz
ftp.ncbi.nlm.nih.gov/pub/taxonomy/accession2taxid/pdb.accession2taxid.gz

Should I use the non-redundant option?

This option is only available for Uniprot or Swissprot selected databases. The non-redundant option merges UniprotKB database fasta entries with same sequence into one entry with concatenated headers. Header seperator SOH (\x01)
You should use it!
