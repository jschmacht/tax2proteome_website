Example 1: Generate protein database using Uniprot database and unchanged rank

Tax2proteome run for a metaproteome sample containing species highly represented in the protein databases

Sample composition: Escherichia coli (taxid: 562), Yerisina pseudotuberculosis (taxid:633), unknown species of the Rhizobium (taxid: 379) and Salmonella (taxid: 590) genus, and some viruses of the Coronaviridae (taxid: 11118) family are assumed.

1. Select taxa for database selection by taxon IDs or name by different methods:

The taxon IDs of species or of a certain rank can be determined here: NCBI Taxonomy

Input by name or taxon ID search:

Input by taxon IDs. Separated by comma, space, semicolon or new line:

Input by upload file. All numbers contained in the file are extracted as taxon IDs:

2. Select Options

Option database selection

Chose one of the following databases: Uniprot, NCBI-nr, Swissprot. Tax2Proteome selects all sequences according the given taxa from one of these databases. Since the sample contains E.coli and Salmonella, which are high represented in the databases, the smaller Uniprot Database is chosen.

Option rank selection

Chose rank, the higher, the more protein sequences are contained in the resulting database. Here option species is chosen, so that the defined rank by user given taxon IDs remains unchanged. For high represented species like Salmonella and E.coli a rank increase is unnecessary and decreases the identification rate.

Option for databases selected from the Uniprot or Swissprot protein databases

Option non-redundant merges all redundant sequences to one with concatenated fasta headers. The search engine matching results are improved by using non-redundant databases, so use it!

3. Control input taxa

Control the selected taxa and ranks in the input table. If taxa with higher rank as specified in options are given by the user, they stay unchanged.

4. Send your request for generating a protein sequence database based on public sequence databases accordingly to your chosen taxa and options.

push the send button

You get your database ID here:

5. Check progress of database creation.

Depending on server workload, the database creation can take up to one day, but mostly it will take only few minutes. You can check the status of your customized database in the progress tab, using the (in 4.) provided database ID.

6. Download your specified database by link.

For database download follow the download link provided by the browser after checking progress returns status 01 - download ready.

Sample composition: Cupriavidus metallidurans (taxid: 119219), Rhodotorula glutinis (taxid: 5535)

For taxa input use one of the following input options: font_download or view_list or note_add

2. Select Options

Option database selection

Chose one of the following databases: Uniprot, NCBI-nr, Swissprot. Tax2Proteome selects all sequences according the given taxa from one of these databases. Since the sample contains only rare species, which are low represented in the databases, the bigger NCBI Database is chosen.

Option rank selection

Chose rank, the higher, the more protein sequences contained in the resulting database.Here option genus is chosen. Since the requested species are only poorly investigated, they have only few entries in the public protein databases. The identification can profit from including sequences of related species.

Options for databases selected from the NCBI-nr protein database

Option 'reduced headers' reduces the NCBI fasta-headers to their accession IDs. This reduces the memory requirement up to 50 %.
